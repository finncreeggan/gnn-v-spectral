## Goal

Build and run the evaluation and analysis pipeline for synthetic graph benchmark experiments. We compare 9 community detection methods (6 spectral + 3 GNN) on SBM and LFR graphs under controlled structural noise, using Adjusted Rand Index (ARI) as the primary metric.

ARI is the right metric because our task is recovering planted community structure, and ARI is invariant to label permutation, making it more appropriate than classification accuracy for graph partition recovery.

This document reflects the **actual implemented codebase** as of March 29, 2026.

## File Formats

### Edge files (`{graph_id}.csv`)
CSV with columns: `src, dst, same_comm, comm_pair`

### Label files (`{graph_id}_labels.npy`)
NumPy int64 array of shape `(n_nodes,)` — planted community labels aligned to node indices 0..n-1.

### Spectra files (`{graph_id}.pt`)
A single PyTorch file per graph containing a dict with 4 tensors:

| Key | Shape | Description |
|-----|-------|-------------|
| `whole_V` | `(n_nodes, n_nodes)` | Full eigenvector matrix of the symmetric normalized Laplacian |
| `whole_evals` | `(n_nodes,)` | Full eigenvalues, sorted ascending |
| `reg_V` | `(n_nodes, n_nodes)` | Eigenvectors of the Tikhonov-regularized Laplacian (L + tau*I) |
| `reg_evals` | `(n_nodes,)` | Regularized eigenvalues |

**Note:** The k-cut eigenspectrum is **not stored on disk**. It is computed on-the-fly from `whole_V` and `whole_evals` using the eigengap heuristic inside `load_graph_data()`.

### Feature files (`{graph_id}_feature_informativeness_{code}_features.npy`)
NumPy float32 array of shape `(n_nodes, 5)` — synthetic node features at a given informativeness level. Generated for Experiment 2 only.

## File Naming Convention

Graph and label files follow the pattern:

```
{graph_id}.csv                    # edges
{graph_id}_labels.npy             # labels
{graph_id}.pt                     # spectra
```

Where `graph_id` encodes all provenance:

```
graph{NNN}_{noise_code}_{noise_type}_{family}
```

Examples:
- `graph001_000_clean_sbm` — base graph 1, clean, SBM
- `graph001_005_random_sbm` — base graph 1, 5% random noise, SBM
- `graph003_045_targeted_betweenness_lfr` — base graph 3, 45% targeted noise, LFR

### Noise Variables

| Variable | Type | Values | Description |
|----------|------|--------|-------------|
| `noise_type` | categorical | `clean`, `random`, `targeted_betweenness` | Perturbation mechanism |
| `noise_code` | string | `000`, `005`, `010`, ..., `045` | Encoded noise fraction |
| `noise_frac` | float | `0.0`, `0.05`, `0.10`, ..., `0.45` | Fraction of edges removed |

Noisy graphs form **true perturbation chains**: for a given base graph, the versions at `noise_code = 005, 010, ..., 045` are descendants of the same clean parent, not independently regenerated. This ensures performance degradation reflects increasing noise, not graph-to-graph variation.

## The GraphData Dataclass

All data loading goes through a central dataclass defined in `data/__init__.py`:

```python
@dataclass
class GraphData():
    graph: BaseData                    # torch_geometric.Data (edge_index, num_nodes)
    graph_id: str

    noise_fraction: float
    num_classes: int
    labels: Int[Tensor, "n_nodes"]

    # Three eigenspectra (precomputed, loaded from .pt file)
    whole_eigenvals: Float[Tensor, "n_nodes"] | None
    whole_eigenspectrum: Float[Tensor, "n_nodes n_nodes"] | None

    kcut_eigenvals: Float[Tensor, "n_eigenvectors"] | None
    kcut_eigenspectrum: Float[Tensor, "n_nodes n_eigenvectors"] | None

    regularized_eigenvals: Float[Tensor, "n_nodes"] | None
    regularized_eigenspectrum: Float[Tensor, "n_nodes n_nodes"] | None

    # Node features (identity matrix if not provided)
    features: Float[Tensor, "n_nodes feature_dim"] | None

    # Transductive 70/15/15 split (generated deterministically via seed)
    train_idx: Int[Tensor, "num_train_nodes"] | None
    val_idx: Int[Tensor, "num_valid_nodes"] | None
    test_idx: Int[Tensor, "num_test_nodes"] | None
```

## The load_graph_data() Function

A single function loads everything from disk into a `GraphData` object:

```python
def load_graph_data(
    metadata_csv: str | Path,    # path to graph_index_{family}.csv
    graph_id: str,               # row identifier
    *,
    features_pt: str | Path | None = None,   # optional .npy or .pt feature file
    seed: int = 0,               # seed for deterministic 70/15/15 split
    dataset_root: str | Path = "data/cache/synthetic",
) -> GraphData
```

**What it does step by step:**

1. Reads `metadata_csv`, looks up the row for `graph_id`
2. Loads edge CSV from `row["edge_path"]` -> builds `torch_geometric.Data`
3. Loads labels from `row["label_path"]` -> `Int[Tensor, "n_nodes"]`
4. Loads spectra from `row["spectra_path"]` -> whole_V, whole_evals, reg_V, reg_evals
5. Computes k-cut eigenspectrum on-the-fly via eigengap heuristic on `whole_evals`
6. Loads features from `features_pt` (`.npy` or `.pt`) or falls back to identity matrix
7. Generates deterministic 70/15/15 train/val/test split using `torch.manual_seed(seed)`
8. Returns a fully populated `GraphData` object

**Key design choice:** Train/val/test splits are generated inside `load_graph_data()` with a fixed seed, not stored as separate files. This guarantees the same split for a given graph across all experiments.

## The Three Eigenspectra

### 1. Whole Eigenspectrum
Full eigendecomposition of the symmetric normalized Laplacian L. Returns all n_nodes eigenvectors and eigenvalues. Used by `whole_lr` and `whole_rf` methods.

### 2. K-cut Eigenspectrum
Bottom-k eigenvectors selected via the **eigengap heuristic**: find the largest gap in eigenvalues starting from index 1 (skipping the trivial zero eigenvalue at index 0). The number of eigenvectors k is determined automatically — it typically corresponds to the number of communities. Used by `kcut_lr` and `kcut_rf` methods.

### 3. Regularized Eigenspectrum
Full eigendecomposition of the Tikhonov-regularized Laplacian: `L_tik = L + tau * I` where `tau = mean(degree)`. This shifts all eigenvalues away from zero, improving numerical stability. Used by `regularized_lr` and `regularized_rf` methods.

## The Nine Classifiers

All methods are registered in `methods/registry.py` under `METHOD_REGISTRY` and share a common interface via `BaseMethod`:

```python
class BaseMethod(abc.ABC):
    def fit(self, data: GraphData, ...) -> Self
    def score(self, data: GraphData, *, use_test_idx: bool = False) -> dict[str, float]
```

`score()` returns `{"ARI": float}`.

### Spectral Methods (6) — Fully Implemented

| Key | Embedding | Classifier | Description |
|-----|-----------|------------|-------------|
| `whole_lr` | Whole eigenspectrum | Logistic Regression | Full spectrum + LR on train_idx |
| `whole_rf` | Whole eigenspectrum | Random Forest | Full spectrum + RF on train_idx |
| `kcut_lr` | K-cut eigenspectrum | Logistic Regression | Bottom-k eigenvectors + LR |
| `kcut_rf` | K-cut eigenspectrum | Random Forest | Bottom-k eigenvectors + RF |
| `regularized_lr` | Regularized eigenspectrum | Logistic Regression | Regularized spectrum + LR |
| `regularized_rf` | Regularized eigenspectrum | Random Forest | Regularized spectrum + RF |

All spectral methods compose an embedding type with a classifier type via the `SpectralMethod` class. The classifier receives `[embeddings || features]` concatenated as input. In Experiment 1 (no informative features), features default to the identity matrix, so the classifier operates on spectral embeddings alone.

The pipeline passes **precomputed embeddings** from `GraphData` to `SpectralMethod.fit()` to avoid recomputing expensive eigendecompositions:

```python
if isinstance(classifier, SpectralMethod):
    embedding_map = {
        "whole": data.whole_eigenspectrum,
        "kcut": data.kcut_eigenspectrum,
        "regularized": data.regularized_eigenspectrum,
    }
    classifier.fit(data, embeddings=embedding_map[classifier.embedding_type])
```

### GNN Methods (3) — Stubs, Implementation Needed

| Key | Architecture | Description |
|-----|-------------|-------------|
| `sgc` | SGConv (k-hop propagation + linear) | Middle ground between spectral and GNN |
| `gcn` | GCNConv (2-layer) | Standard message-passing GNN |
| `gat` | GATv2Conv (multi-head attention) | Attention-based GNN |

These are currently stub implementations that return empty dicts from `score()`. They need full PyTorch implementations using `torch_geometric` layers. Each should:
1. Build a `torch.nn.Module` from `ExperimentConfig` (hidden_dim, num_layers, lr, epochs, dropout, etc.)
2. Train with cross-entropy on `data.train_idx` for `config.epochs` steps
3. Predict community labels via argmax of output logits
4. Score with ARI on val/test_idx

## Evaluation Protocol

The evaluation is **explicitly transductive**: every method sees the full graph structure, but only training-node labels are revealed during fitting. Test labels remain hidden until final scoring.

- **Train:** 70% of nodes — used for fitting
- **Validation:** 15% of nodes — used for hyperparameter selection (Optuna, when wired)
- **Test:** 15% of nodes — used for final ARI reporting

The independent unit of uncertainty is the **base graph**, not the node split. Final uncertainty bands reflect variation across the 5 base graphs at each condition.

## Part I. Structural Noise-Sweep Experiment

### Objective

Quantify how ARI changes across the 9 models as structural noise increases in four settings:

1. SBM + random edge deletion
2. SBM + targeted edge deletion (betweenness centrality)
3. LFR + random edge deletion
4. LFR + targeted edge deletion

### Experiment Table

`build_metadata_tables.py` reads the graph_index CSVs, adds a `spectra_path` column, saves enriched graph_index CSVs, then creates `structural_noise_experiment_table.csv` with columns:

```
graph_id, family, base_graph_id, seed,
structural_noise_type, structural_noise_code, structural_noise_frac,
edge_path, label_path, spectra_path,
n_nodes, num_edges, num_communities, ...
```

### Pipeline Step by Step

1. `build_metadata_tables.py` builds the experiment table
2. For each row in the experiment table, for each of 9 model keys:
   a. `load_graph_data()` loads graph, labels, spectra, generates split
   b. `_get_model()` creates the classifier with `ExperimentConfig`
   c. `classifier.fit(data)` — spectral methods receive precomputed embeddings
   d. `classifier.score(data)` on val split -> `best_validation_ari`
   e. `classifier.score(data, use_test_idx=True)` -> `test_ari`
   f. One row appended to raw results CSV (incremental writes for fault tolerance)
3. Failed runs logged to `failed_runs.csv` with error message and timestamp

### Raw Results CSV

```
graph_id, family, base_graph_id,
structural_noise_type, structural_noise_code, structural_noise_frac,
model, split_id, optuna_n_trials,
best_validation_ari, test_ari, best_params_json
```

### Summary Tables

**Graph-level summary:** one row per (graph_id, model) — mean val/test ARI across splits.

**Condition-level summary:** averaged across the 5 base graphs for each family x noise_type x noise_code x model condition. Includes mean and std for both validation and test ARI.

### Figures

**4 joint comparison plots** (one per family x noise_type): each shows one line per model, x-axis = structural_noise_frac (0.00 to 0.45), y-axis = mean test ARI with error bars across 5 base graphs.

**36 per-model diagnostic plots** (4 settings x 9 models): transparent graph-level ARI scatter, prominent mean points, fitted trend line.

## Part II. Feature-Informativeness Experiment

### Objective

Measure how model performance changes as the **informativeness of node features decreases** while structural noise is held fixed.

### Selected Conditions

Three structural-noise levels x two noise types x two families = 12 structural settings:

- `structural_noise_code = 015` (low noise)
- `structural_noise_code = 030` (medium noise)
- `structural_noise_code = 045` (high noise)

Each condition has 5 base graph descendants, giving **60 selected graphs**.

### Feature Generation

For each graph, 11 feature matrices are generated at informativeness levels alpha = 1.0, 0.9, ..., 0.0:

$$X_i = \alpha \cdot f_5(y_i) + (1 - \alpha) \cdot \epsilon_i, \quad \epsilon_i \sim \mathcal{N}(0, I_5)$$

where $y_i$ is the planted class label and $f_5$ is a fixed 5th-degree nonlinear mapping using trigonometric polynomials:

```python
prototypes = [sin(theta)^5, cos(theta)^3*sin(theta)^2,
              sin(2*theta)^5, cos(3*theta)^5,
              sin(theta)^2*cos(2*theta)^3]
```

Angles are evenly spaced: $\theta_k = 2\pi k / n_{communities}$.

Each feature matrix is saved as a `.npy` file with a reproducible seed stored in the experiment table.

60 graphs x 11 levels = **660 feature matrices** total.

### Feature-Informativeness Variables

| Variable | Type | Values |
|----------|------|--------|
| `feature_informativeness_code` | string | `100`, `090`, ..., `000` |
| `feature_informativeness_frac` | float | `1.00`, `0.90`, ..., `0.00` |
| `feature_noise_frac` | float | `0.00`, `0.10`, ..., `1.00` |

### Pipeline Step by Step

1. `build_metadata_tables.py` creates `feature_informativeness_experiment_table.csv`
2. `generate_feature_informativeness.py` generates all 660 `.npy` feature matrices
3. For each (graph, feature_level, model) triple:
   a. `load_graph_data()` loads graph + spectra + features (via `features_pt=`)
   b. Fit and score as in Experiment 1
   c. Append row to raw results CSV

Spectral methods receive features but their classifiers concatenate `[embeddings || features]`, so they benefit from informative features. GNNs use features as input node attributes directly.

### Figures

**12 joint plots** (one per family x noise_type x noise_code): x-axis = feature_informativeness_frac (1.0 at left, 0.0 at right), validation and test curves for all 9 models.

**108 per-model plots** (12 settings x 9 models): graph-level scatter + mean trend lines.
